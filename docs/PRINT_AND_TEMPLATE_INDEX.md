# Print Format & HTML Template Code – Full Index

Single reference for **where** all print and invoice template code lives, and how to ensure it works in **production** before push.

---

## 1. Backend – PDF generation (QuestPDF, no HTML-to-PDF in production)

| Location | Purpose |
|----------|--------|
| **`backend/HexaBill.Api/Modules/Billing/PdfService.cs`** | Main PDF service. `GenerateInvoicePdfAsync(SaleDto, format)` supports **A4, A5, 80mm, 58mm**. A5/80mm/58mm use QuestPDF-only; A4 uses QuestPDF (optionally loads HTML for future use but **does not** convert HTML to PDF in production). |
| **`backend/HexaBill.Api/Modules/Billing/IPdfService.cs`** | Interface: `GenerateInvoicePdfAsync(SaleDto sale, string format = "A4")`. |
| **`backend/HexaBill.Api/Modules/Billing/SaleService.cs`** | `GenerateInvoicePdfAsync(int saleId, int tenantId, string? format)` loads sale, builds DTO, calls `_pdfService.GenerateInvoicePdfAsync(saleDto, formatNormalized)`. Normalizes format to A4 | A5 | 80mm | 58mm. |
| **`backend/HexaBill.Api/Modules/Billing/SalesController.cs`** | `GET /sales/{id}/pdf?format=A4|A5|80mm|58mm` – reads `format` from query, validates, returns PDF file. |

**Format handling in PdfService (summary):**

- **A4:** QuestPDF layout (page A4, margin 5mm), company header, items table, subtotal/VAT/round-off/total, footer. Optional: load HTML from `InvoiceTemplateService` or `Templates/invoice-template.html` for future use; **actual PDF is always QuestPDF**.  
- **A5:** `GenerateInvoicePdfA5` – compact QuestPDF layout.  
- **80mm:** `GenerateInvoicePdf80mm` – thermal receipt QuestPDF.  
- **58mm:** `GenerateInvoicePdf58mm` – narrow thermal QuestPDF.

## 2. Backend – HTML template (for A4 preview / future HTML-to-PDF)

| Location | Purpose |
|----------|--------|
| **`backend/HexaBill.Api/Modules/Billing/InvoiceTemplateService.cs`** | Renders HTML: `RenderActiveTemplateAsync` (DB template or fallback file), `RenderTemplateHtmlAsync` (replace placeholders: company, invoice_no, date, customer, items, subtotal, vat_amount, grand_total, amount_in_words, logo). Default file path: `Templates/invoice-template-hd.html`. |
| **`backend/HexaBill.Api/Modules/Billing/InvoiceTemplatesController.cs`** | CRUD for invoice templates (get/create/update, preview with sample data). |
| **`backend/HexaBill.Api/Modules/Billing/PdfService.cs`** | For A4 only: loads `templateHtml` via `_templateService.RenderActiveTemplateAsync` or from `Templates/invoice-template.html`. This HTML is **not** converted to PDF in production (that would require DinkToPdf/PuppeteerSharp); A4 PDF is generated by QuestPDF. `GeneratePdfFromHtmlTemplateAsync` exists but throws `NotImplementedException` – reserved for future HTML-to-PDF. |
| **Template files (optional)** | `backend/HexaBill.Api/Templates/invoice-template.html`, `invoice-template-hd.html` – used as fallback when no active DB template. Placeholders: `{{company_name_en}}`, `{{company_trn}}`, `{{invoice_no}}`, `{{date}}`, `{{customer_name}}`, `{{customer_trn}}`, `{{subtotal}}`, `{{vat_amount}}`, `{{grand_total}}`, `{{amount_in_words}}`, `{{company_logo_img}}` / `{{company_logo_data_uri}}`, plus items table. |

**Important:** Production invoice PDFs are **all QuestPDF**. HTML is for template editing/preview and future HTML-to-PDF if you add a library later.

## 3. Frontend – Where print is triggered and format is passed

| Location | Behavior |
|----------|----------|
| **`frontend/hexabill-ui/src/services/index.js`** | `salesAPI.getInvoicePdf(id, options)` – `options.format` sent as query `format=A4|A5|80mm|58mm`. Default (no format) = A4. |
| **`frontend/hexabill-ui/src/components/PrintOptionsModal.jsx`** | After Save in POS: user picks **A4 / A5 / 80mm / 58mm**, then Print. Calls `salesAPI.getInvoicePdf(saleId, { format })`, opens blob in new tab and triggers print. |
| **`frontend/hexabill-ui/src/pages/company/PosPage.jsx`** | After save, shows success modal with **Print** → opens `PrintOptionsModal` (format selector). Other flows (e.g. quick print without modal) use `getInvoicePdf(saleId)` → A4 default. |
| **`frontend/hexabill-ui/src/pages/company/SalesLedgerPage.jsx`** | Download invoice PDF: `salesAPI.getInvoicePdf(entry.saleId)` – no format → A4. |
| **`frontend/hexabill-ui/src/pages/company/CustomerLedgerPage.jsx`** | `salesAPI.getInvoicePdf(invoiceId)` / `getInvoicePdf(selectedInvoiceForView)` – no format → A4. |
| **`frontend/hexabill-ui/src/pages/company/PaymentsPage.jsx`** | Download: `salesAPI.getInvoicePdf(payment.saleId)` – no format → A4. |
| **`frontend/hexabill-ui/src/components/InvoicePreviewModal.jsx`** | Preview/Download: `salesAPI.getInvoicePdf(saleId)` – no format → A4. |

So: **only the POS “Print” path (PrintOptionsModal) sends a format**. All other invoice PDFs use default A4, which is correct for reports and downloads.

## 5. Production readiness – before push checklist

Use this **before pushing** to ensure all print and template paths work in production.

### Backend

- [ ] **API format param:** `GET /sales/{id}/pdf?format=` accepts `A4`, `A5`, `80mm`, `58mm`; invalid/missing defaults to A4.
- [ ] **PdfService:** No unhandled exception when generating A4, A5, 80mm, 58mm (test with at least one invoice that has items + VAT).
- [ ] **SQLite/PostgreSQL:** DB has required columns (e.g. `Sales.RoundOff`, `Payments.SaleReturnId`) so PDF generation and sales ledger don’t throw “no such column” (see `docs/SQLITE_MISSING_COLUMNS_FIX.md`).
- [ ] **Fonts:** QuestPDF runs without font errors in production (Arabic if used); optional fonts in `Fonts` folder or system fallback.
- [ ] **Templates folder:** If you use file fallback, `Templates/invoice-template.html` or `invoice-template-hd.html` exist in the deployed app directory (or rely only on DB templates).

### Frontend

- [ ] **POS flow:** Save invoice → success modal → Print → PrintOptionsModal shows A4, A5, 80mm, 58mm → choosing one and clicking Print opens PDF in new tab and print dialog (or download if pop-up blocked).
- [ ] **getInvoicePdf:** All call sites that need a specific format pass `options.format`; others correctly get A4 by default (Sales Ledger, Customer Ledger, Payments, Invoice Preview).
- [ ] **Errors:** Failed PDF (e.g. 500) shows a clear message (e.g. toast) and does not leave the app in a broken state.

### Data and compliance

- [ ] **Same numbers in all formats:** For one saved invoice, A4, A5, 80mm, 58mm show the same Subtotal, VAT, Grand Total (no recalculation in template).
- [ ] **Gulf VAT:** All formats include company name, TRN, invoice number, date, items, VAT rate/amount, total, currency (see `PRINT_FORMATS_TESTING.md`).

### Quick smoke test (before push)

1. Start backend and frontend (local or staging).  
2. Create one invoice in POS (2–3 items, VAT), Save.  
3. Click Print → choose A4 → Print → confirm PDF opens and numbers match.  
4. Repeat for A5, 80mm, 58mm (same invoice).  
5. From Sales Ledger, download PDF for that invoice → should be A4 and same totals.  
6. Check backend logs for no “no such column” or PDF generation errors.

## 6. Summary

- **Print format code:** Backend in `PdfService.cs` (A4/A5/80mm/58mm) and `SalesController.cs` (query param); frontend in `PrintOptionsModal.jsx` and `services/index.js` `getInvoicePdf(id, { format })`.  
- **HTML template code:** `InvoiceTemplateService.cs` (render with placeholders), optional files under `Templates/`. Used for preview and future HTML-to-PDF; **production PDFs are generated with QuestPDF only.**  
- **Before push:** Run the production checklist above and the smoke test so all formats work and data is consistent.
